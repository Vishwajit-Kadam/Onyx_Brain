# Work Contracts

Each advanced autonomous run writes a work contract under its workspace.

The contract records:

- the original user goal
- the interpreted goal type
- required deliverables
- assumptions made without follow-up questions
- limitations such as markdown-only decks
- safety boundaries
- the done definition used to decide whether the run is complete

Full-bounded mode proceeds without asking for confirmation, but it still records the contract so the final report can be audited against it.
