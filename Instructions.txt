3. ROLLBACK SYSTEM
==================================================

Create:

src/agency/rollback.rs

Goal:
Undo the last failed or risky operation.

Add command:

cargo run -- rollback latest

Behavior:
1. Find latest rollback-capable journal entry.
2. If it has a rollback_ref, restore previous state.
3. If it belongs to a snapshot, restore snapshot.
4. Mark journal entry as RolledBack.
5. Write rollback report.

Add:

cargo run -- rollback --project calc_cli latest

Rollback only within the named project.

Rollback report:

RollbackReport {
    rollback_id: String,
    target_entry_id: String,
    project_name: Option<String>,
    actions_reverted: u64,
    files_restored: u64,
    status: String,
    errors: Vec<String>,
    created_at: DateTime<Utc>,
}

Store:

data/logs/rollback_report_{timestamp}.json

Rules:
- Never rollback outside sandbox.
- Never delete user files outside sandbox.
- If uncertain, refuse rollback and explain.
- Do not rollback cargo target artifacts unless they were explicitly journaled.

FileEditTransaction {

}

- Failed
- RolledBack

For every file edit:
1. Validate path is inside sandbox.
2. Create backup.
3. Write to temp file.
4. Validate temp file is UTF-8.
5. Replace original atomically if possible.
6. Commit transaction.
7. Journal entry updated.

If failure:
- restore backup
- mark Failed
- include error

Store transaction logs:

data/transactions/{transaction_id}.json

Add command:

cargo run -- transactions

Shows recent transactions.
